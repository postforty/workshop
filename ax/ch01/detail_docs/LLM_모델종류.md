# LLM 모델의 종류

## 개요
LLM은 사용 방식에 따라 크게 **클라우드 API 모델**과 **오픈소스 모델**로 나뉩니다. 각각의 특징과 장단점을 이해하면 상황에 맞는 모델을 선택할 수 있습니다.

## 1. 클라우드 API 모델

### 특징
인터넷을 통해 제공되는 서비스로, 웹 브라우저나 API를 통해 접근합니다.

### 대표 모델

| 모델 | 제공사 | 주요 특징 | 웹사이트 |
|------|--------|----------|----------|
| **ChatGPT** | OpenAI | 가장 대중적, GPT-4 기반 | https://chatgpt.com |
| **Gemini** | Google | 구글 서비스 연동, 멀티모달 | https://gemini.google.com |
| **Claude** | Anthropic | 긴 문맥 처리, 안전성 강조 | https://claude.ai |
| **Copilot** | Microsoft | Office 통합, 엔터프라이즈 | https://copilot.microsoft.com |

### 장점
- ✅ **설치 불필요**: 웹 브라우저만 있으면 즉시 사용 가능
- ✅ **최신 모델**: 지속적으로 업데이트되는 최신 버전 사용
- ✅ **높은 성능**: 강력한 서버 인프라로 빠른 응답 속도
- ✅ **유지보수 불필요**: 모델 관리는 제공사가 담당

### 단점
- ❌ **비용 발생**: 무료 플랜 제한, 대량 사용 시 비용 증가
- ❌ **인터넷 필수**: 오프라인 환경에서 사용 불가
- ❌ **데이터 프라이버시**: 입력 데이터가 외부 서버로 전송됨
- ❌ **커스터마이징 제한**: 모델 자체를 수정하거나 조정할 수 없음

### 적합한 사용 사례
- 개인 사용자의 일상적인 질문 및 작업
- 프로토타입 개발 및 테스트
- 최신 성능이 중요한 경우
- IT 인프라 구축이 어려운 경우

## 2. 오픈소스 모델

### 특징
소스 코드가 공개되어 있어 누구나 다운로드하여 로컬 환경에서 실행할 수 있습니다.

### 대표 모델

| 모델 | 제공사 | 주요 특징 | 라이선스 |
|------|--------|----------|----------|
| **Llama 3** | Meta | 높은 성능, 다양한 크기 | Meta Llama 3 (조건부 무료) |
| **Gemma** | Google | 경량화, 빠른 실행 | Gemma Terms (무료) |
| **Qwen** | Alibaba | 다국어 지원, 코딩 특화 | Apache 2.0 (무료) |
| **DeepSeek** | DeepSeek AI | 코딩 특화, 논리적 추론(Reasoning) 우수 | MIT (무료) |
| **EXAONE** | LG AI Research | 한국어 특화, 기업용 | CC BY-NC-SA 4.0 (비상업적 사용만 무료) |
| **SOLAR** | Upstage | 한국어 성능 우수, 경량화 | Apache 2.0 (무료) |

### 장점
- ✅ **무료 사용**: 라이선스 조건만 준수하면 비용 없음
- ✅ **로컬 실행**: 인터넷 없이 오프라인 환경에서 사용 가능
- ✅ **데이터 통제**: 모든 데이터가 로컬에 유지되어 프라이버시 보장
- ✅ **커스터마이징**: 특정 도메인에 맞게 모델을 재학습(Fine-tuning) 가능

### 단점
- ❌ **하드웨어 요구사항**: 고성능 GPU 필요 (VRAM 8GB 이상 권장)
- ❌ **설치 및 설정**: 기술적 지식 필요
- ❌ **성능 차이**: 클라우드 최신 모델 대비 성능이 낮을 수 있음
- ❌ **유지보수**: 모델 업데이트 및 관리를 직접 해야 함

### 적합한 사용 사례
- 민감한 데이터를 다루는 기업 환경
- 오프라인 환경에서의 사용
- 특정 도메인에 특화된 모델 개발
- 비용 절감이 중요한 대규모 프로젝트

## 비교 요약

| 구분 | 클라우드 API 모델 | 오픈소스 모델 |
|------|------------------|--------------|
| **비용** | 유료 (무료 플랜 제한) | 무료 (하드웨어 비용 별도) |
| **설치** | 불필요 | 필요 |
| **성능** | 매우 높음 | 중~높음 |
| **인터넷** | 필수 | 불필요 |
| **프라이버시** | 외부 서버 전송 | 로컬 유지 |
| **커스터마이징** | 제한적 | 자유로움 |
| **기술 난이도** | 낮음 | 중~높음 |

## 선택 가이드

### 클라우드 API 모델을 선택하세요:
- 빠르게 시작하고 싶을 때
- 최신 성능이 중요할 때
- IT 인프라가 부족할 때
- 개인 또는 소규모 프로젝트

### 오픈소스 모델을 선택하세요:
- 데이터 프라이버시가 중요할 때
- 장기적으로 비용을 절감하고 싶을 때
- 모델을 커스터마이징하고 싶을 때
- 오프라인 환경에서 사용해야 할 때
